# 思维框架提炼方法论

> 从原始信息到可运行的心智模型，核心方法论

## 一、心智模型识别的三重验证

一个论点要被认定为「心智模型」而非「随口一说」，必须通过三重验证：

### 验证1: 跨域复现
同一个思维框架出现在此人讨论的至少2个不同领域。

例：纳瓦尔的「杠杆」概念——
- 在财富创造中谈杠杆（代码、媒体、资本、劳动力）
- 在个人成长中谈杠杆（特定知识 + 杠杆 = 复利）
- 在职业选择中谈杠杆（选择有杠杆效应的工作）
→ 跨3个领域复现 → 这是真正的心智模型

### 验证2: 有生成力
用这个模型可以推断此人对新问题的可能立场。

例：如果芒格的「逆向思维」是心智模型——
- 面对「如何成功」→ 他会先想「如何确保失败」
- 面对「如何投资」→ 他会先想「如何亏光所有钱」
→ 能生成新推断 → 这是真正的心智模型

### 验证3: 有排他性
不是所有聪明人都会这样想，这个模型体现了此人的独特视角。

例：「反脆弱」是塔勒布的，不是所有人都这样看世界
→ 有区分度 → 这是值得提炼的心智模型

**如果一个观点只通过了1-2重验证** → 降级为「决策启发式」而非「心智模型」
**如果一个观点0重验证都没过** → 可能只是此人在特定场景说的话，不纳入

## 二、六维度结构化提炼

把一个活人拆成6个可编程的维度。**关键区分：捕捉的是HOW they think，不是WHAT they said。**

### 2.1 世界观滤镜（怎么看世界）

= 同一件事，不同人看到的东西不一样。要抓的是这个人看任何问题时**第一反应关注什么**。

**操作步骤**：

1. **扫描**：逐个读取调研文件，列出所有候选（此人反复表达的观点、自创术语、核心主张）。通常会得到15-30个候选
2. **筛选**：对每个候选执行三重验证（见上文第一节）
   - 三重通过 → 世界观滤镜
   - 仅1-2重 → 降级为决策直觉
   - 0重 → 丢弃
3. **排序取舍**：按排他性强度排序（越独特越靠前），取3-5个。宁少勿多——3个深刻的比10个浅薄的强
4. **记录格式**：每个滤镜记录——名称、一句话白话描述、来源证据（≥2个场景）、这副眼镜怎么用、什么时候这副眼镜会骗你

**示例**：
- 芒格的眼镜：「先看激励结构」→ 遇到任何组织/人的行为，第一反应是"谁在这里面有利益？利益结构怎么扭曲了行为？"
- 塔勒布的眼镜：「先看尾巴」→ 遇到任何风险评估，第一反应是"最坏情况有多坏？概率模型能不能覆盖极端场景？"
- 张小龙的眼镜：「先看用户会不会烦」→ 遇到任何产品功能，第一反应是"这个功能加上去之后用户的负担是什么？"

### 2.2 决策直觉（怎么做决定）

= 此人做判断时的快速规则，可表述为「如果…就…」，必须有真实案例支撑。

**提取5-10条**，每条包含：
- 规则本身（一句话）
- 真实案例（他在什么场景下用了这条规则）
- 反面案例（他违反这条规则的时候发生了什么，如果有的话）

**示例**：
- 巴菲特：「如果10分钟看不懂，就不投」→ 案例：拒绝投科技股长达20年
- 乔布斯：「如果需要用户手册，就是设计失败」→ 案例：初代iPhone取消了实体键盘
- 芒格：「如果一个决策可以反过来想，先想怎么会死」→ 案例：投资前先列出所有可能亏损的方式

### 2.3 嘴替特征（怎么说话）

= 一段话发出来，别人一看就知道是谁说的。**要求具体到可以模仿的程度。**

| 维度 | 不要写这种 | 要写这种 |
|------|-----------|---------|
| 句式 | 「语言犀利」 | 「爱用反问句收尾，先说一个荒谬的反面来衬托正面」 |
| 口头禅 | 「有独特表达」 | 「高频使用'skin in the game'、把不喜欢的人叫'fragilista'」 |
| 节奏 | 「表达有力」 | 「先抛结论→举一个极端反例→回到结论，三段式，从不铺垫」 |
| 怼人方式 | 「不留情面」 | 「直接说对方是'intellectual yet idiot'，从不对事不对人」 |
| 幽默 | 「有幽默感」 | 「冷幽默+自嘲，经常用自己的投资亏损当笑话开头」 |
| 确定性 | 「表达自信」 | 「从不说"我觉得"，直接断言，但会加一句"当然我可能是错的"」 |

#### 表达DNA量化方法

从此人的长文/演讲中随机抽取20个段落，统计：

| 维度 | 测量方式 |
|------|---------|
| 平均句长 | 字数/句数 |
| 疑问句比例 | 疑问句数/总句数 |
| 类比密度 | 类比数/千字 |
| 第一人称使用率 | 「我」出现频率 |
| 确定性语气比例 | 「一定」「显然」vs「也许」「可能」 |
| 转折频率 | 「但是」「然而」「不过」/千字 |

风格标签维度：

```
正式 ←→ 口语
抽象 ←→ 具体
谨慎 ←→ 断言
学术 ←→ 通俗
长句 ←→ 短句
铺垫型 ←→ 结论先行
数据驱动 ←→ 叙事驱动
```

禁忌词和口癖：
- 此人从不用的词 → 生成Agent时也不用
- 此人的口癖/高频表达 → 适度使用（太多变成模仿秀）

### 2.4 雷区和底线（什么不做）

= 一个人不做的事比做的事更能定义他。

提取内容：
- **价值观底线**：3-5条核心价值排序（比如「宁可少赚钱也不碰看不懂的东西」）
- **公开怼过的东西**：他明确反对什么？（反模式）
- **拒绝过的诱惑**：什么机会他主动放弃了？为什么？
- **内在矛盾**：真人不是逻辑自洽的，矛盾才真实。比如「主张长期主义但也做过短期投机」——这些张力恰恰是深度的来源

### 2.5 来路（经历塑造了什么）

= 不知道一个人经历过什么，就不可能理解他为什么这样。

**不是流水账年表**，而是**只保留塑造了2.1-2.4的那些经历**：

- 哪件事让他形成了某个世界观滤镜？
- 哪次失败改变了他的决策规则？
- 谁影响了他？他的思想师承是什么？
- 什么转折让他变成了现在的表达风格？

格式：3-7个关键节点，每个节点写清楚「发生了什么 → 塑造了什么」。

### 2.6 老实话（做不到什么）

必须明确写出的局限：
- 哪些方面调研信息不够，结论是猜的
- 这个人的哪些能力是文字化不了的（比如现场读人的直觉、即兴反应的节奏感）
- 公开表达 vs 真实想法可能有多大差距
- 信息截止到调研时间点
- 至少3条具体局限，不是只有「不能替代本人」这种废话

## 三、矛盾处理原则

矛盾是人格的核心特征，不是需要修复的Bug。

### 三种矛盾类型

1. **时间性矛盾**（观点演化）
   - 此人早期说A，后来说B
   - 处理：记录演化轨迹，标注「早期」「近期」
   - 在SOUL.md中以「近期观点」为主，但提及演化

2. **领域性矛盾**（不同场景不同规则）
   - 此人在工作中主张X，在生活中主张Y
   - 处理：分领域记录，不强求统一
   - 这恰恰是深度的来源

3. **本质性张力**（价值观内在冲突）
   - 例：既追求自由又重视纪律
   - 处理：明确记录为「核心张力」
   - 这通常是此人最有意思的部分

### 错误的处理方式
- ❌ 选一边忽略另一边
- ❌ 编一个调和的解释
- ❌ 假装矛盾不存在

## 四、信息不足时的处理

| 情况 | 处理 |
|------|------|
| 某个维度公开信息很少 | 在SOUL.md中标注「信息不足，此维度为推测」 |
| 只有二手信息 | 降低置信度，标注为「据[来源]报道」 |
| 信息相互矛盾且无法判断 | 并列呈现，让用户自行判断 |
| 此人刻意不公开某方面 | 尊重边界，在SOUL.md中注明「此人对这个话题保持沉默」 |

**关键规则**：宁可生成一个诚实标注了局限的60分Agent，也不要生成一个看起来完美但实际上在编造的90分Agent。

## 五、人物Agent vs 主题Agent的区别

| 维度 | 人物Agent | 主题Agent |
|------|----------|----------|
| 核心 | 一个人的思维方式 | 一个领域的思维工具箱 |
| 心智模型来源 | 主要来自一个人 | 综合多人视角 |
| 表达风格 | 模拟此人的表达 | 中性但专业 |
| 矛盾处理 | 保留个人内在矛盾 | 呈现流派间分歧 |
| 验证方式 | 对比此人已知立场 | 对比领域内共识 |

## 六、质量自检清单

生成Agent人设文件后，用以下问题自检：

### 心智模型（世界观滤镜）
- [ ] 每个模型都有至少2个不同领域的证据？
- [ ] 模型数量在3-7个之间？（太少=太浅，太多=没提炼）
- [ ] 每个模型都有明确的应用场景和局限（失效条件）？
- [ ] 模型之间有张力但不矛盾？

### 表达DNA（嘴替特征）
- [ ] 读起来有辨识度，不像通用AI？
- [ ] 没有过度模仿变成caricature？
- [ ] 抓住了核心特征而非表面模仿？
- [ ] 具体到可以模仿的程度（不只是「语言犀利」这种）？
- [ ] **包含「反AI输出模式」提取结果？（此人怎么不说话）**
- [ ] **提取结果已转化为可执行的禁止规则，而非仅仅是描述？**

### 决策启发式（决策直觉）
- [ ] 每条规则都有具体案例支撑？
- [ ] 可以被新情况触发（不只适用于原始案例）？
- [ ] 有反面案例（违反规则时的后果）？

### 回答工作流（Agentic Protocol）
- [ ] 包含问题分类（需要事实/纯框架/混合）？
- [ ] 研究维度来自此人的世界观滤镜？
- [ ] 明确要求用工具获取事实，不可跳过？

### 诚实边界（老实话）
- [ ] 明确写了做不到什么？（至少3条具体局限）
- [ ] 标注了信息源和调研时间？
- [ ] 承认了信息不足的维度？
- [ ] 不是只有「不能替代本人」这种废话？

### 雷区底线
- [ ] 包含内在矛盾？（至少1对）
- [ ] 观点不是高度一致的？（太一致=太假）

### 整体
- [ ] 用此人的眼睛看一个新问题，能得到有价值的视角？
- [ ] 不是此人原话的拼凑，而是框架的运行？
- [ ] 删掉名字后，还能认出这是谁的思维方式？
- [ ] 一手来源占比 >50%？
