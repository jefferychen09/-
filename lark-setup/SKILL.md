# 飞书 CLI 安装引导与自然语言翻译

> `lark-cli` 是飞书官方提供的命令行工具，让人类和 AI Agent 都能在终端中操作飞书。

本技能分为两大部分：
1. **自动安装** — **强制**自动完成 CLI 的安装与凭证检查，授权通过管家「连接」面板完成
2. **自然语言翻译** — 将用户的自然语言请求翻译为具体的 `lark-cli` 命令并执行

> ⚠️ **重要**：当本技能被激活时，你**必须立即自动执行**第一部分的全部检查与安装步骤，**不要询问用户是否要安装**，直接执行命令。只有在遇到无法自动解决的错误时才向用户报告。

---

## 第一部分：自动安装（强制执行）

> **强制规则**：本部分的所有步骤**必须自动执行**，不要向用户询问"是否要安装"或"是否继续"。直接运行命令，只在失败时报告错误。

### 步骤 1：检查 Node.js 版本（强制）

立即执行：

```bash
node --version
```

- Node.js >= 18 → 继续下一步
- 未安装或版本过低 → **停止并告知用户**：
  > Node.js 18 或更高版本是必需的前置依赖，请先安装后重试。
  > - macOS/Linux 推荐使用 [nvm](https://github.com/nvm-sh/nvm)：`nvm install 18 && nvm use 18`
  > - Windows 推荐使用 [nvm-windows](https://github.com/coreybutler/nvm-windows) 或直接从 [Node.js 官网](https://nodejs.org/) 下载安装

### 步骤 2：检查并安装 lark-cli（强制）

立即执行：

```bash
lark-cli --version
```

- 已安装 → 记录版本号，继续下一步
- 未安装（命令不存在）→ **立即自动安装，不要询问用户**：

**macOS / Linux：**

```bash
export NPM_CONFIG_PREFIX="$QCLAW_NPM_GLOBAL_PREFIX" && npm install -g @larksuite/cli --force
```

**Windows：**

```powershell
$env:NPM_CONFIG_PREFIX=$env:QCLAW_NPM_GLOBAL_PREFIX; npm install -g @larksuite/cli --force
```

> 如果上述包名安装失败，尝试从 GitHub 安装：
> ```bash
> export NPM_CONFIG_PREFIX="$QCLAW_NPM_GLOBAL_PREFIX" && npm install -g larksuite/cli --force
> ```

安装完成后验证：

```bash
lark-cli --version
```

> ⚠️ **必须**在 `npm install -g` 前设置 `NPM_CONFIG_PREFIX` 为 `QCLAW_NPM_GLOBAL_PREFIX` 的值。

### 步骤 3：检查凭证状态（强制）

检查飞书是否已通过管家连接面板完成授权：

**macOS / Linux：**

```bash
ls ~/.lark-cli/config.json 2>/dev/null
```

**Windows：**

```powershell
Test-Path "$env:USERPROFILE\.lark-cli\config.json"
```

判断逻辑：
- `config.json` 存在 → **已授权**，直接跳到步骤 5
- `config.json` 不存在 → 执行步骤 4

### 步骤 4：引导用户通过管家连接面板授权（仅凭证缺失时执行）

> ⚠️ **禁止执行 `lark-cli auth login` 命令**。飞书授权已由管家应用的连接面板接管，通过 OAuth 2.0 设备流直接调用飞书 HTTP API 完成，凭证会自动写入 `~/.lark-cli/config.json` 和系统 Keychain，与 lark-cli 完全兼容。

**直接告知用户**：
> 飞书尚未连接。请在管家左侧的「连接」面板中找到「飞书」，点击「连接」按钮完成授权。授权完成后我会自动检测到凭证，你可以再次让我执行飞书操作。

**不要**尝试执行 `lark-cli auth login --recommend` 或任何其他 `lark-cli auth` 命令。

### 步骤 5：验证凭证可用（强制）

验证 lark-cli 可以正常使用已有凭证：

```bash
lark-cli auth status
```

- 返回已认证状态（identity 为 "user"）→ 安装成功，展示功能概览卡片
- 返回未认证或 token 过期 → 提示用户在管家「连接」面板中断开飞书并重新连接

## 第二部分：自然语言翻译为 CLI 命令

### 翻译总则

当用户用自然语言描述飞书操作时，按以下规则翻译为 CLI 命令：

1. **识别意图** → 判断属于哪个品类（calendar/im/doc/drive/sheets/base/task/mail/contact/wiki）
2. **确定方法** → 匹配具体的工具方法
3. **提取参数** → 从自然语言中提取参数值
4. **执行命令** → 运行 `lark-cli <品类> <方法> [参数]`
5. **解读结果** → 将返回值翻译为用户友好的自然语言

## 注意事项

1. **自动安装**：本技能激活后**必须自动执行**安装检查和安装步骤，不要询问用户"是否安装"
2. **前置条件**：使用前需确保 Node.js >= 18 已安装
3. **凭证安全**：飞书凭证存储在 OS 原生 Keychain 中（macOS Keychain / Windows Credential Manager），配置文件位于 `~/.lark-cli/config.json`
4. **凭证检测**：执行任何操作前，先检查 `~/.lark-cli/config.json` 是否存在
5. **授权方式**：飞书授权**统一通过管家「连接」面板完成**，**禁止**执行 `lark-cli auth login` 命令。管家会通过 OAuth 2.0 设备流直接调用飞书 API，凭证自动写入与 lark-cli 兼容的格式
6. **重新授权**：如需更换账号，告知用户在管家「连接」面板中先断开飞书，再重新连接
7. **破坏性操作**：删除文件、取消日程等操作执行前**必须**向用户确认
8. **错误重试**：遇到 HTTP 错误时主动重试，最多 3 次
